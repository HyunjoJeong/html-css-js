이벤트와 이벤트 처리기

이벤트란

이벤트란 웹 문서 영역 안에서만 이루어지는 동작을 의미한다.
(따라서, 뒤로 가기 등 브라우저 창 위에서 클릭하는 것은 이벤트가 아니다.)

마우스 이벤트

종류설명
click사용자가 html 요소를 클릭할 때 이벤트 발생
dblclick사용자가 html 요소를 클릭할 때 이벤트 발생
mousedown사용자가 요소 위에서 마우스 버튼을 눌렀을 때 이벤트 발생
mousemove사용자가 요소 위에서 마우스 포인터를 움직일 때 이벤트 발생
mouseover사용자가 요소 위로 마우스 포인터를 옮길 때 이벤트 발생
mouseout사용자가 마우스 포인터를 요소에서 벗어날 때 이벤트 발생
mouseup사용자가 요소 위에 놓인 마우스 버튼에서 손을 땔 때 이벤트 발생

키보드 이벤트

종류설명
keydown사용자가 키를 누르는 동안 이벤트 발생
keypress사용자가 키를 눌렀을 때 이벤트 발생
keyup사용자가 키에서 손을 땔 때 이벤트 발생

문서 로딩 이벤트

종류설명
abort문서가 완전히 로딩되기 전 불러오기를 멈췄을 때 이벤트 발생
error문서가 정확히 로딩되지 않았을 때 이벤트 발생
load문서 로딩이 끝나면 이벤트 발생
resize문서 화면 크기가 바뀌었을 때 이벤트 발생
scroll문서 화면에서 스크롤 되었을 때 이벤트 발생
unload문서에서 벗어날 때 이벤트 발생

폼 이벤트

종류설명
blur폼 요소에 포커스를 잃었을 때 이벤트 발생
change목록이나 체크 상태 등이 변경되면 이벤트 발생(input, select, textarea 태그에서 사용)
focus폼 요소에 포커스가 놓였을 때 이벤트 발생(label, select, textarea, button 태그에서 사용)
reset폼이 리셋되었을 때 이벤트 발생
submitsubmit 버튼을 클릭했을 때 이벤트 발생

이벤트 처리기(Event Handler)

1. html 태그에 직접 연결하는 방법:

- 문법: <태그 on이벤트명 = "함수명">

2. DOM을 이용한 이벤트 처리기

- querySelector, querySelectorAll등을 이용해 요소를 연결후 on이벤트명 = 함수;
- ex : document.querySelector("#hi").onclick = () => alert("하이");

* 참고: document.querySelector("#hi").onclick = alert("하이"); 와 같이 작성하면 브라우저 렌더링시 자동으로 클릭되는 이상한 오류가 있음.(onclick만 그런 듯?)
