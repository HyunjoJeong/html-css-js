헤더 영역

검색 창, 사이트 메뉴를 삽입한다.

네비게이션 영역

  • 위치 1로 이동
  • 위치 2로 이동
  • 위치 3로 이동

헤더나 푸터, 사이드 바 내에서도 사용 가능 (웹 문서의 위치에 영향 받지 않는다.)

메인 영역

메뉴, 사이드바, 로고처럼 페이지마다 똑같이 들어가는 정보는 넣을 수 없고,
웹 문서마다 다르게 보여주는 내용으로 구성

아티클 영역

블로그의 포스트나 뉴스의 기사처럼 독립된 웹 콘텐츠 항목을 의미.
여러개 사용도 가능하고, 내부에 section 태그를 넣을 수도 있음.

섹션 영역

아티클 태그와 비슷해 보이지만 다름.
몇개의 콘텐츠를 묶는 용도로 사용.

사이드바 영역(aside)

본문 내용 외에 왼쪽 오른쪽 혹은 아래에 사이드바를 생성

푸터 영역

웹사이트 맨 아래에 위치
사이트 정보나, 저작권, 연락처 등을 넣음.
header 태그나 section, article 등도 푸터 내부에 사용할 수 있음.

디비전 영역

가장 예전부터 사용하던 방법.
사실상 div 태그만 사용해도 됨.
그러나 코드 가독성에서의 이점 및 화면 낭독기 등 보조 기기에서의 차이 등이 있어 semantic tag를 사용하는 것이 더 좋을 것.
